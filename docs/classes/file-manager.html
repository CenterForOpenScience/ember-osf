Copy a file or folder. Convenience method for move with options.copy == true.

Parameters:

  • file File

    file model to copy.

  • targetFolder File

    Where to copy the file, a file model with isFolder == true.

  • [options] Object optional

    Options hash

    • [query] Object optional

      Key-value hash of query parameters to add to the request URL.

    • [data] Object optional

      Payload to be sent.

      • [rename] String optional
        If specified, also rename the file to the given name.
      • [resource] String optional
        Optional node ID. If specified, move the file to that node.
      • [provider] String optional
        Optional provider name. If specified, move the file to that provider.
      • [conflict='replace'] String optional
        Specifies what to do if a file of the same name already exists in the target folder. If 'keep', rename this file to avoid conflict. If replace, the existing file is destroyed.

Returns:

Promise:

Promise that resolves to the the new file model or rejects with an error message.

move

  • file
  • targetFolder
  • [options]
)
Promise

Defined in addon/services/file-manager.js:212

Move (or copy) a file or folder

Parameters:

  • file File

    file model to move.

  • targetFolder File

    Where to move the file, a file model with isFolder == true.

  • [options] Object optional

    Options hash

    • [query] Object optional

      Key-value hash of query parameters to add to the request URL.

    • [data] Object optional

      Payload to be sent.

      • [rename] String optional
        If specified, also rename the file to the given name.
      • [resource] String optional
        Optional node ID. If specified, move the file to that node.
      • [provider] String optional
        Optional provider name. If specified, move the file to that provider.
      • [action='move'] String optional
        Either 'move' or 'copy'.
      • [conflict='replace'] String optional
        Specifies what to do if a file of the same name already exists in the target folder. If 'keep', rename this file to avoid conflict. If replace, the existing file is destroyed.

Returns:

Promise:

Promise that resolves to the the updated (or newly created) file model or rejects with an error message.
